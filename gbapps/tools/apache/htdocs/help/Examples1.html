The following example demonstrates examples of how rules can be created using the Glassbeam Rules and Alerts App.

  1. Is a recent VLAN firmware upgrade causing an addressing, routing configuration problem, or is the change impacting the buffer, memory, authentication, encryption, or compression?

Item

Description / Action to Take

Description

Understand the VLAN configuration inside the log sections to discover if there are configuration issues

Logic (Evaluation Criteria)

UPPER({VLAN_compgrp.VLAN_compgrp_str}) LIKE '%NO VLAN COMPRESSION GROUPS EXIST%'

Alert Condition (Initiate action recommendation criteria) 

techsupportinfo : Show VLAN compr-group all, Vlan optimization bugs. Current State is- {VLAN_compgrp.VLAN_compgrp_str}

Immediate Actions

Discover if a recent firmware update for optimization is causing the configuration conflict resulting in compression, routing, addressing, or such issues. 

Check for the corresponding MRD in the bug database.

Evaluate whether the development action taken addresses the optimization bug release


  1. Let’s set the criteria and discover if there are uncorrectable errors in DIMMs from the var.log files.

Item

Description / Action to Take

Description

The DIMMs with uncorrectable error will be disabled if DIMM blacklisting is enabled or if the DIMM fails on reboot during BIOS POST and OS does see that memory. UCSM operState will be Inoperable for this DIMM in this case.


Uncorrectable DIMMs cannot be fixed. In addition, it is impossible for the application or operating system to continue execution. Also, a problem with DIMMs can cause a server to fail to boot or cause the server to run below its capabilities.

Logic (Evaluation Criteria)

UPPER({varlogs.Event Text}) LIKE '%#0X7E%'  OR UPPER({varlogs.Event Text}) LIKE '%#0X02%'  AND { varlogs.State}<>"" AND { varlogs.Source}<>"" AND { varlogs.Label}<>""

Alert Condition (Initiate action recommendation criteria)

Var log: Current State is { varlogs.Source}  { varlogs.Event Text} { varlogs.Label}   { varlogs.State}

Immediate Actions

Increase the Java heap size, see internal knowledge base article: 1090
  1. Let’s check if a there’s Link Failure in a networking equipment installed at a customer site. 

Item

Description / Action to Take

Description

Link failure can be caused by the following reasons:  General link failure occurs due to various reasons such as  loss of signal (LOS) or not operational (NOS). Other probable causes can be high bit error rate, too many interrupts, cable is disconnected, BB_credit buffers overflow, hardware recoverable errors, connected device rebooted (F ports only), connected linecard rebooted (ISL only), and so on. 

Logic (Evaluation Criteria)

UPPER({show Interface.Interface}_status) LIKE '%ERRDISABLED%'  AND {show Interface.Interface}<>""

Alert Condition (Initiate action recommendation criteria)

Show Interface: Interface status is not-connected or link failure or errdisabled Current State is-Interface ={show Interface.Interface}, Status = {show Interface.Status}

Immediate Actions

Switch interfaces that are experiencing issues.


  1. Typical power modules in any computing unit will over its lifetime show the ‘Power Supply Failure’ status. Let’s set a criterion to assess and report on the current status of a power module from the error log file. 

Item

Description / Action to Take

Description

Power Supply Failure

Logic (Evaluation Criteria)

UPPER({UCSM Show fault detail.Code}) LIKE '%F0373%'  AND {UCSM Show fault detail.Description}<>""

Alert Condition (Initiate action recommendation criteria)

Sam Techsupport Info: Power supply failure for module10909. Current state is  {UCSM Show fault detail.Description} Cause: {UCSM Show fault detail.UCSM Show fault detail_Cause}

Immediate Actions

Refer to KB 2994: UCS Chassis Power Supply Errors Indicate PSU Degraded And Failed
  1. A common message and troubleshooting area is the ‘I / O module Fault’. Let’s use the following logic to evaluate if there’s a possible I/O module fault showing signs of shutting down. 

Item

Description / Action to Take

Description

Discover whether there is an I / O module fault and alert relevant personnel. 

Logic (Evaluation Criteria)

UPPER((!({I/O Module Status.SPA State}) LIKE '%EMPTY%'  AND UPPER({I/O Module Status.SPASubStat}) LIKE '%MISSING%'  AND UPPER({I/O Module Status.SPB State}) LIKE '%EMPTY%'  AND UPPER({I/O Module Status.SPBSubStat}) LIKE '%MISSING%' ) AND UPPER(!({I/O Module Status.SPA State}) LIKE '%PRESENT%'  AND UPPER({I/O Module Status.SPASubStat}) LIKE '%GOOD%'  AND UPPER({I/O Module Status.SPB State}) LIKE '%PRESENT%'  AND UPPER({I/O Module Status.SPBSubStat}) LIKE '%GOOD%' ))

Alert Condition (Initiate action recommendation criteria)

I/O module UCSM Show fault detail: Module {I/O Module Status.Mod}  spa has state {I/O Module Status.SPA State} and has sub state {I/O Module Status.SPASubStat}. Module spb has state {I/O Module Status.SPB State} and has sub state {I/O Module Status.SPBSubStat}

Immediate Actions

Alert to shutdown UCSM module to field staff at site A.

  6. VERSION( {WC-Version.version_new1} ) >= '%11.0.30.00.420%'
  7. VERSION( {WC-Version.version_new1} ) <= '%11.0.30.00.420%' AND VERSION( {WC-Version.version_new1} ) >= '%11.0.30.00.400%'
