About System Groups, Roles and Users

Role:  A Role links Users, Products and Applications together. I.e. you can create a role to give access to a list of users, for a specific set of products and its associated applications. 

For example, to assign users to access the Dashboards app only for a product called ‘Hiwi’,

  • Create a role 
  • Associate this role to ‘Hiwi’ product
  • Configure dashboards as a feature for the role 
  • Associate users to this role 

Once associated, the users that are part of this role can access Dashboards application for the product Hiwi.  By default, the Admin role is available. This administrator role has the controls to invoke, revoke, and provide access to apps based on the number of user licenses as a customer you have purchased.

Note: A user can be associated to only one role. I.e. the same user cannot be part of multiple roles. 

User:  To access any Glassbeam application every user needs to have a unique login ID created in Glassbeam. These login IDs are associated to specific roles depending on the access that needs to be provided to the user. You can create 2 types of users:

  1. External User:  These are users who are outside of your organization, typically your business end users, who want to view Health Check reports created by Glassbeam Professional Services for your business. External users are mapped to External Roles. When a role is set as External, the only application that is available for that role is Health Check.  
  2. Internal User:  All users within your organization are called internal users. Internal users have access to applications of Glassbeam limited by their role.

System Group:  System Group is a collection of devices and/or other System Groups.  System Group is relevant only for businesses which have Health Check feature and clinsight application available for their use.

Health check is a feature created by Glassbeam Professional Services team to view your business devices/systems analytics.  Typically, this application can be made available to your business end users as a value-added service.  These end users are called External Users and they are associated to System Groups.  Internal users, however, can be either restricted to one particular System Group or be given access to all the device data in the m/p/s.

External users can access Clinsights application if the configuration is such.  For access to Clinsights please contact support@glassbeam.com.  

System Groups can be part of alert subscription list in Rules and Alerts application.  Click here to know more about subscription list.  

For an external user, the data in the application is restricted to devices that are associated to the End Customer Group to which the external user belongs.  To support Health Check application, Glassbeam’s admin console facilitates the following: 

  • Creation of System Group 
  • Restrict the System Group access to specific devices
  • Embed a System Group in another System Group
  • Associate external user to a System Group

Ex:  Glassbeam’s customer Hi-Wi uses Glassbeam Application with Health Check configuration.  Hi-Wi has a list of end users called Customer1, Customer2, and Customer3 who need access to HealthCheck Application.  System Group is created by associating it to device IDs concerning the group.  External role is created with Health Check configuration.  Customer1, Customer2, and Customer3 are created as external users with associations to External Role and System Group.

Roles are classified as follows: 

  • External - External Roles have only Health Check feature permission by default.  
  • Internal – Admin has the liberty to select feature permissions or Internal Roles. 

Users are classified as follows: 

  • External – are associated with external roles and System Group. 
  • Internal – are associated with internal roles. 

Now, Hi-Wi would configure their user groups as described below to meet the above requirement.

Step 1:

  • Create System Groups called System Group 1 and System Group 2.
  • Associate System Group 1 with Device Ids 001, 002, 003.
  • Associate System Group 2 with Device Ids 004, 005, 006.

Step 2:

  • Create Role1 as an external role with Health Check as the only feature by default.

Step 3:

  • Create external users Customer1, Customer2, and Customer3.  
  • Associate the above external users to Role1 and one of the System Groups - System Group 1 or System Group 2.
