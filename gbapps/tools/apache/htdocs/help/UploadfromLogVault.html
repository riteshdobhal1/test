Upload from Log Vault

You can upload one or more logs by selecting those logs from LogVault assuming the logs you need for testing are already uploaded in Glassbeam.

To upload logs from LogVault for testing rules, click on “Upload from LogVault”. The Glassbeam’sLogVault page is displayed. Use the filter functionalities of LogVault and zero in on the Log bundles that you want to use for testing rules. Select one or more log Bundles for testing.

Note: The more log bundles you select, the more time it will take for testing as it has to parse all the selected bundles one by one. So, select as few bundles as required for testing.


Once uploaded, the below screen displays the processing on the log(s) you just uploaded.

At this point, you can return to the Rules List page to create a new rule or stay on the Rule Test History page to move on to the next step.


Once the processing is complete, the View Results link becomes active. Click the View Results link.


Test Results section displays the number of times in the log file (s) that the alert condition was met. If the count is more than zero, then clicking on the count displays the alert messages, which can be verified to confirm that the rule has been configured correctly. If the count is 0 and you were expecting the rule to be triggered, then go back and check the Rule Logic. 


From the dialog displayed, note the number of times the rule condition was satisfied. 

In addition, if you have set the email reporting option, you will get a message similar to the one displayed as sample below:


If the result is what you expected, then the testing is complete. If the result was not what you expected, then go back to your rules list, edit the rule that you just created and check the logic to see if there is an issue with the rule. Once you are happy with the results of the rule, the rule is ready to be deployed in production.

  1. Click the Enable button to deploy the rule immediately. Once the rule is enabled,the rule is automatically checked against every log bundle that Glassbeam parses.

If you choose not to enable the Rule, then the rules gets listed in the Rule list landing page in the draft state. You can enable the rule anytime. To enable the rule in the Rule List page, select the rule and click the enable button
