Manage E-mail Templates:

E-mail templates are created in order to:

  • specify the format of the E-mail that needs to be sent when an alert has to be sent as an E-mail.
  • specify the list of e-mail addresses that need to receive this alert.

The e-mail templates must be associated with rules so that the alerts are e-mailed

 to people specified in the template.

Manage API Templates:

API templates allow Admin users to add:

  • APIs that can be called when a specific alert occurs.  You will need to provide:
    • The details of the API to be called.
    • The payload data that needs to be sent to the API.

Such a configuration, for example, can be used to open a ticket in your ticketing system automatically when a specific alert occurs.

Base templates contain the base URL for the API along with the authentication required to access the API.  Only an admin user can create base templates.

API Template provides additional details that is required to call the API and is wrapped on top of the base template.  

Payload data in an API template can be fixed values or variables.  Variables are data that come from the configured rule.

Manage Category:

Categories help in the following:

  • Identify the purpose of a rule.
  • Group multiple rules for similar purposes.

The categories must be associated with rules in order for the rules to be categorized.
