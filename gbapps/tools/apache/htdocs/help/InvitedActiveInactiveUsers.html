Invited/Active/Inactive Users

When a user is created for the first time, the user is in INVITED status.  An email is sent to the user with a link which is used by the new user to set the password and subsequently login.  Below is the sample of the email being sent to the new user who is in the INVITED status.


The user is activated and moved to ACTIVE status after the following steps are followed:

  • Invited user sets the password using the link provided in the ‘Welcome’ email (shown above).
  • Invited user logs in to the Glassbeam Application for the first time using newly set password.  


You cannot activate/deactivate a user in INVITED status.

You can inactivate an ACTIVE user by using the Inactivate button (shown above). The user's state changes to INACTIVE when inactivated.   This is particularly useful when: 

  • You do not want to delete the user entirely but want to reuse the licenses for other users in your organization.
  • You wish to make a user temporarily non-operational.

A user whose status is set to INACTIVE is not counted as part of the total number of licenses available for your organization based on the user’s roles/features.

You can activate an INACTIVE user by using the Activate button (shown above).  The user’s state changes to ACTIVE when activated.  A user whose status is set to ACTIVE is now part of the total number of licences available for your organization based on the user’s roles/features.

Note: 

An admin user status cannot be set to INACTIVE.  The admin user will always remain in the ACTIVE state.
