Saving the view

Saving Section views helps users apply the saved customized views on other log files. For example, to identify a specific issue related to an authentication problem, a user creates a view of all relevant sections, filtered and formatted to make troubleshooting easy. If this view is saved and shared, then every time any user comes across a similar problem, instead of recreating the view, the user can use the saved view and apply it to the log file for troubleshooting.

To save a section view, do this:

  1. Make changes as required to the section views for a log bundle. See the above sections on customizing the section views. 
  2. Once complete, click the Save View icon from the main navigation bar.  The Save View dialog displays.



  1. Enter an appropriate name and description. Change the Access option to Private if you do not want this view to be accessible publically. By default, your view is set as private.
  2. Click Submit to complete.

The view is now saved. 

  1. To see the saved view. Go to Select View above the Select Section panel. Your view is now available in the My Views tab. 

Note: The Selected Sections area displays the name and its corresponding attributes for the section. The Attributes table displays the details of each of the selected sections. 

Note: When a selected section is modified by transposing the column data or applying numeric filters, then the corresponding section is indicated by an icon in the Attributes table.
